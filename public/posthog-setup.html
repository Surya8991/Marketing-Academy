Setup Guide

PostHog Analytics — Setup

Wire up free usage analytics for your Marketing Academy site. Track which lessons get viewed, completed, and quizzed.

Free tier — 1M events/month, no CC required
Package — posthog-js
Env vars — 2 (both NEXT_PUBLIC_*)

0 of 11 steps complete

1
PostHog Account & Project ~3 min
Sign up at posthog.com
Go to posthog.com → Get started free. No credit card. Free tier is 1M events/month — more than enough for personal use.
Choose a region and create your project
During onboarding pick US or EU data residency. The region determines your API host:
US → https://us.i.posthog.com EU → https://eu.i.posthog.com
Name the project Marketing Academy and select Web as the platform.
Copy your Project API Key
After project creation, PostHog shows a snippet with your key: phc_xxxxxxxxxxxxxxxxxxxxxxxx. Copy it — this is your NEXT_PUBLIC_POSTHOG_KEY.
Also findable later: Settings → Project → Project API Key.
2
Environment Variables ~2 min
Add to .env.local (local dev)
In the project root, add to your local .env.local file:
NEXT_PUBLIC_POSTHOG_KEY=phc_your_key_here NEXT_PUBLIC_POSTHOG_HOST=https://us.i.posthog.com
Change us to eu if you picked EU region.
Add to Vercel environment variables
Vercel dashboard → your project → Settings → Environment Variables. Add both, set to Production + Preview + Development:
NEXT_PUBLIC_POSTHOG_KEY
phc_your_key_here
NEXT_PUBLIC_POSTHOG_HOST
https://us.i.posthog.com
Then redeploy the project.
3
Install & Wire Up in Code ~5 min
Install posthog-js
Run in the project root:
npm install posthog-js
Create src/components/PostHogProvider.tsx
Create a new file with this exact content:
"use client"; import posthog from "posthog-js"; import { PostHogProvider as PHProvider } from "posthog-js/react"; import { useEffect } from "react"; export function PostHogProvider({ children }: { children: React.ReactNode }) { useEffect(() => { if (!process.env.NEXT_PUBLIC_POSTHOG_KEY) return; posthog.init(process.env.NEXT_PUBLIC_POSTHOG_KEY, { api_host: process.env.NEXT_PUBLIC_POSTHOG_HOST ?? "https://us.i.posthog.com", capture_pageview: false, capture_pageleave: true, persistence: "localStorage", }); }, []); return <PHProvider client={posthog}>{children}</PHProvider>; }
ℹ capture_pageview: false — we'll fire page views manually via LessonViewTracker so they include lesson metadata (category, slug, level).
Wrap the root layout in src/app/layout.tsx
Add the import and wrap the <body> children:
// Add at top of layout.tsx: import { PostHogProvider } from "@/components/PostHogProvider"; // Inside <body>, wrap {children}: <PostHogProvider> {/* existing Nav, children, Footer etc */} </PostHogProvider>
⚠ layout.tsx must remain a Server Component — PostHogProvider is the "use client" boundary. Do NOT add "use client" to layout.tsx.
Extend src/components/LessonViewTracker.tsx to fire events
Add a posthog.capture call inside the existing useEffect:
import posthog from "posthog-js"; // Inside the useEffect that fires on pathname change: posthog.capture("lesson_viewed", { category, slug, title, // pass from lesson page props if available level, // "Beginner" | "Intermediate" | "Advanced" });
ℹ The component already tracks the route — just add the capture call alongside whatever it already does (analytics, recently-viewed, etc).

5
Verify It Works ~2 min
Run dev server and open a lesson
npm run dev
Navigate to any lesson page on localhost.
Confirm events appear in PostHog Live Events
In PostHog dashboard → Activity → Live events. Within a few seconds you should see lesson_viewed appear with your lesson's category and slug as properties.
