Setup Guide

Cloudflare KV — Cloud Sync Setup

Follow these steps to activate push/pull sync in your Marketing Academy Settings page.

0 of 14 steps complete

1
Cloudflare Account & KV Namespace ~3 min
Sign in to Cloudflare
Go to dash.cloudflare.com — create a free account if needed. No credit card required.
Open Workers & Pages → KV
In the left sidebar click Workers & Pages, then click the KV tab. If Workers isn't visible, click "Explore Products" and enable it.
Create a KV namespace named ma-progress
Click Create namespace. Enter the name exactly:
ma-progress
The name is just a label — the Namespace ID (shown after creation) is what goes in your env vars.
Copy the Namespace ID
After creation, the namespace appears in the list with a long hex ID like a1b2c3d4e5f6… — copy it and save somewhere. This is your CF_KV_NAMESPACE_ID.
2
Get Your Account ID ~1 min
Find your Account ID
Click Workers & Pages in the sidebar. On the right side under Account details, you'll see a 32-character hex Account ID. Copy it — this is your CF_ACCOUNT_ID.
Also visible in the right sidebar of any domain's Overview page.
3
Create an API Token ~3 min
Open API Tokens
Top-right of the Cloudflare dashboard → click your avatar / profile icon → My Profile → click the API Tokens tab → click Create Token.
Use the "Edit Cloudflare Workers" template
Scroll to find the Edit Cloudflare Workers template and click Use template. This pre-fills the correct permissions including KV write access.
Verify KV permission, then create
Under Permissions confirm you see: Account → Workers KV Storage → Edit. Scroll down → Continue to summary → Create Token.
Copy the token — shown only once
Cloudflare shows the token once on this screen. Copy it now and save it. This is your CF_KV_API_TOKEN.
⚠ If you navigate away without copying, you'll need to delete and recreate the token.
4
Generate a Sync Secret ~1 min
Generate a random secret
This protects your sync API endpoint. Run in terminal:
openssl rand -hex 32
Save the output — you'll paste it into Vercel twice as SYNC_SECRET and NEXT_PUBLIC_SYNC_SECRET.
No terminal? Use any strong 32+ character random string from a password manager.
5
Add Environment Variables to Vercel ~3 min
Open Vercel → your project → Settings → Environment Variables
Go to vercel.com/dashboard → click the Marketing Academy project → Settings tab → Environment Variables in the left sidebar.
Add all 5 environment variables
Add each key, paste the value, set environments to Production + Preview + Development, then click Save:
CF_ACCOUNT_ID
← from Step 5
CF_KV_NAMESPACE_ID
← from Step 4
CF_KV_API_TOKEN
← from Step 9
SYNC_SECRET
← your random string
NEXT_PUBLIC_SYNC_SECRET
← same random string
⚠ SYNC_SECRET and NEXT_PUBLIC_SYNC_SECRET must be the exact same value.
Redeploy the project
Go to the Deployments tab → three dots on the latest deployment → Redeploy. Env vars only take effect after a new deploy.
6
Test It ~1 min
Push from your main device
Go to your site → click the ⚙ gear icon in the nav → scroll to Cloud Sync → click ↑ Push to cloud. Should show: "Saved to cloud at HH:MM:SS".
Pull on another device
On a second device or incognito browser → open your site → Settings → ↓ Pull from cloud. The page reloads with your progress restored.
✓ Done. From now on: push after each session, pull when switching devices.
